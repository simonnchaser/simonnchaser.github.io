## 1. 문제는 어떻게 주어졌는가

이 개념이 궁금해진 계기는 MySQL 문제를 읽다가였다.

문제에서는 `ECOLI_DATA` 테이블이 주어지고, 각 대장균 개체의 `GENOTYPE` 값이 정수로 저장되어 있었다. 그리고 설명에서는 특정 개체가 `1번 형질`, `2번 형질`, `3번 형질`, `4번 형질`을 가졌는지 여부를 기준으로 조건을 읽어야 했다.

예시 설명은 대략 이런 흐름이었다.

- 어떤 개체의 `GENOTYPE` 값이 `8`이다.
- 이를 이진수로 나타내면 `1000₂`이다.
- 따라서 이 개체는 `4번 형질`을 가진다.

처음 읽을 때 가장 헷갈렸던 부분은 바로 여기였다.

- 왜 `8`을 `1000₂`로 바꾸는 것까지는 이해되는데
- 그 다음에 갑자기 `4번 형질`이라는 표현으로 넘어가는가

즉 숫자의 크기를 보고 있는 것인지, 아니면 이진수의 자리 위치를 보고 있는 것인지가 처음에는 명확하지 않았다.

## 2. 왜 비트마스크가 헷갈렸는가

문제를 읽다 보면 `GENOTYPE` 값이 정수로 주어지는데, 설명에서는 갑자기 `1번 형질`, `2번 형질`, `3번 형질`처럼 말하는 경우가 있다.

예를 들어 `1000₂`를 보고 `4번 형질만 가지고 있다`고 설명하면 처음에는 조금 이상하게 느껴질 수 있다.

왜냐하면 눈에는 `1000`이라는 숫자가 보이는데, 설명은 숫자 자체보다 각 자리의 위치를 기준으로 해석하고 있기 때문이다.

## 3. 비트마스크(Bitmask)란 무엇인가

비트마스크는 여러 개의 상태나 속성을 하나의 정수에 비트 단위로 저장하는 방식이다.

즉 하나의 숫자 안에 여러 개의 on/off 정보를 넣는 개념이다.

각 비트는 보통 다음처럼 해석한다.

- `1`이면 해당 속성을 가진다
- `0`이면 해당 속성을 가지지 않는다

이 방식은 형질, 권한, 옵션, 상태값 등을 저장할 때 자주 사용된다.

## 4. 핵심은 숫자 크기가 아니라 비트 위치이다

비트마스크에서 중요한 것은 숫자의 모양보다도 각 비트의 위치이다.

오른쪽부터 보면 각 위치는 다음 값을 가진다.

| 비트 위치 | 값 | 사람이 붙인 번호 예시 |
| --- | --- | --- |
| 0번째 비트 | $2^0 = 1$ | 1번 형질 |
| 1번째 비트 | $2^1 = 2$ | 2번 형질 |
| 2번째 비트 | $2^2 = 4$ | 3번 형질 |
| 3번째 비트 | $2^3 = 8$ | 4번 형질 |

여기서 중요한 점은 `4번 형질`이 숫자 `4`를 뜻하는 것이 아니라, 오른쪽에서 네 번째 위치의 비트를 뜻한다는 점이다.

## 5. 왜 `1000₂`는 4번 형질인가

이진수 `1000₂`를 보자.

오른쪽부터 비트를 세면 다음과 같다.

| 자리 | 값 |
| --- | --- |
| 4번째 자리 | 1 |
| 3번째 자리 | 0 |
| 2번째 자리 | 0 |
| 1번째 자리 | 0 |

이걸 비트 위치 기준으로 다시 보면 다음과 같다.

| 비트 위치 | 값 |
| --- | --- |
| 3번째 비트 | 1 |
| 2번째 비트 | 0 |
| 1번째 비트 | 0 |
| 0번째 비트 | 0 |

즉 `1`이 켜져 있는 곳은 `3번째 비트` 하나뿐이다.

사람이 읽기 쉽게 1부터 번호를 붙이면:

- 0번째 비트 -> 1번 형질
- 1번째 비트 -> 2번 형질
- 2번째 비트 -> 3번 형질
- 3번째 비트 -> 4번 형질

따라서 `1000₂`는 `4번 형질만 가지고 있다`고 말하는 것이다.

## 6. 형질 번호와 실제 값은 다를 수 있다

이 부분이 가장 많이 헷갈린다.

예를 들면:

- 1번 형질의 실제 값은 $1$
- 2번 형질의 실제 값은 $2$
- 3번 형질의 실제 값은 $4$
- 4번 형질의 실제 값은 $8$

즉 형질 번호는 비트의 순서를 말하고, 실제 값은 그 비트가 차지하는 자리값이다.

그래서 `4번 형질`은 숫자 `4`가 아니라 값 `8`인 비트를 뜻한다.

## 7. 문제 지문은 이렇게 해석하면 된다

이런 유형의 문제에서 `GENOTYPE`이 정수로 주어졌다면 보통 다음 순서로 읽으면 된다.

1. 먼저 정수를 이진수 구조로 생각한다.
2. 오른쪽부터 각 비트의 위치를 확인한다.
3. 어떤 위치가 `1`인지 확인한다.
4. 문제에서 말하는 `n번 형질`을 그 비트 위치에 대응시킨다.

즉 문제 지문에서 형질 번호를 말할 때는 대개 숫자 그 자체를 뜻하는 것이 아니라, 비트 위치를 번호처럼 부르고 있는 것이다.

## 8. 비트마스크를 이렇게 읽으면 편하다

비트마스크 관련 문제를 읽을 때는 다음처럼 생각하면 이해가 훨씬 쉬워진다.

1. 정수를 이진수로 본다.
2. 오른쪽부터 비트 위치를 센다.
3. 각 위치의 `1`은 하나의 속성이 있다고 해석한다.
4. 문제에서 말하는 `n번 형질`은 보통 `n번째 위치`에 대응한다고 본다.

즉 숫자를 계산 대상으로만 보지 말고, 여러 개의 스위치가 들어 있는 묶음이라고 생각하면 된다.

## 9. 왜 이런 표현 방식을 쓰는가

비트마스크는 여러 상태를 한 번에 저장할 수 있어서 효율적이다.

예를 들어 형질 4개를 각각 따로 저장하지 않고, 하나의 숫자로 표현할 수 있다.

그래서 문제 풀이뿐 아니라 실제 프로그래밍에서도 자주 등장한다.

- 사용자 권한 관리
- 게임 상태 저장
- 옵션 on/off 관리
- 형질이나 속성 조합 표현

## 10. 정리

`1000₂`를 보고 `4번 형질`이라고 말하는 것은 이 문제만의 특수 규칙이 아니라, 비트마스크를 읽는 일반적인 방식이다.

핵심은 다음과 같다.

- 비트마스크는 여러 속성을 하나의 정수에 저장하는 방식이다.
- 각 형질은 숫자 자체가 아니라 비트 위치에 대응한다.
- 어떤 위치의 비트가 `1`이면 그 형질을 가지고 있다고 해석한다.
- 따라서 `1000₂`는 오른쪽에서 네 번째 위치만 켜져 있으므로 `4번 형질`이다.
