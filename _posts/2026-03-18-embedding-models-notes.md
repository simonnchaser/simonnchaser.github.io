텍스트 마이닝과 NLP를 공부하다 보면 `Word2Vec`, `GloVe`, `FastText`, `BERT` 같은 여러 임베딩 모델을 만나게 된다.  
이 모델들은 모두 텍스트를 숫자 벡터로 표현한다는 공통점이 있지만, 어떤 단위를 임베딩하는지, 문맥을 반영하는지, 어떤 장단점이 있는지에서 차이가 있다.

이 글은 대표적인 임베딩 모델들을 한 번에 정리하면서, 왜 `one-hot encoding`에서 분산 표현으로 넘어가게 되었는지, 그리고 정적 임베딩에서 문맥 임베딩으로 어떻게 발전했는지를 흐름 중심으로 정리한 노트다.

특히 아래 질문에 답이 되도록 정리해보려고 했다.

- 임베딩은 왜 필요한가
- `Word2Vec`, `GloVe`, `FastText`는 무엇이 다른가
- `ELMo`, `BERT`, `Sentence-BERT`는 어떤 전환점을 만들었는가
- 실제 NLP 작업에서는 어떤 기준으로 모델을 골라야 하는가

## 1. One-hot Encoding의 한계와 분산 표현의 등장

임베딩 모델을 이해하려면 먼저 왜 `one-hot encoding`만으로는 부족했는지를 보는 것이 좋다.

`one-hot encoding`은 단어를 단순한 구분용 벡터로 표현하는 방식이다.  
예를 들어 vocabulary에 단어가 5개 있다고 하면, 각 단어는 하나의 위치만 1이고 나머지는 모두 0인 벡터로 표현된다.

예:

- `cat` -> `[1, 0, 0, 0, 0]`
- `dog` -> `[0, 1, 0, 0, 0]`
- `apple` -> `[0, 0, 1, 0, 0]`

이 방식은 "어떤 단어인가"를 구분하는 데는 유용하지만, 여러 한계를 가진다.

### 1.1 One-hot Encoding의 한계

#### 1. 벡터가 너무 희소하다

단어 수가 많아질수록 벡터 길이는 매우 길어지지만, 실제 값은 대부분 0이 된다.  
즉, 비효율적인 희소 벡터(sparse vector)가 된다.

#### 2. 차원이 vocabulary 크기만큼 커진다

만약 단어장이 10,000개면 각 단어 벡터 길이도 10,000이 된다.  
말뭉치가 커질수록 계산량과 메모리 부담이 커진다.

#### 3. 단어 사이의 의미 관계를 표현하지 못한다

`cat`과 `dog`는 의미상 비슷한 단어지만 one-hot 벡터에서는 단지 서로 다른 위치에 1이 있을 뿐이다.  
즉:

- `cat`과 `dog`
- `cat`과 `apple`

이 모두 똑같이 "서로 다른 단어"로만 보인다.

컴퓨터 입장에서는 어떤 단어들이 비슷한지, 관련 있는지, 같은 범주인지 알 수 없다.

#### 4. 일반화가 어렵다

비슷한 단어들끼리 정보를 공유하지 못하기 때문에, 모델이 단어 간 공통 패턴을 학습하기 어렵다.

즉, one-hot encoding은 단어의 정체성(identity)은 표현하지만 의미(meaning)는 표현하지 못한다.

### 1.2 분산 표현이란 무엇인가

이 한계를 극복하기 위해 나온 개념이 `분산 표현(distributed representation)`이다.

분산 표현은 단어를 긴 0/1 벡터가 아니라, 상대적으로 짧은 길이의 dense vector로 표현한다.  
즉, 여러 차원에 걸친 실수값 조합으로 단어의 의미를 표현한다.

예:

- `cat` -> `[0.21, -0.44, 0.80, ...]`
- `dog` -> `[0.18, -0.40, 0.77, ...]`

이런 식으로 표현되면, 의미가 비슷한 단어끼리 벡터도 비슷해질 수 있다.

### 1.3 왜 "분산" 표현이라고 부르는가

one-hot encoding에서는 단어 정보가 딱 한 위치에만 들어 있다.  
즉, 하나의 단어가 하나의 차원에만 의존한다.

반면 분산 표현에서는 단어의 의미가 여러 차원에 나뉘어 담긴다.  
예를 들어 어떤 차원은 동물성, 어떤 차원은 감정, 어떤 차원은 행동성과 같은 특성을 부분적으로 반영할 수 있다.

즉, 단어의 의미가 하나의 값에 몰려 있는 것이 아니라 여러 차원에 분산되어 표현되기 때문에 분산 표현이라고 부른다.

### 1.4 분산 표현의 장점

분산 표현은 다음과 같은 장점을 가진다.

- dense vector라서 계산 효율이 좋음
- 저차원 표현이 가능함
- 단어 간 유사도를 계산할 수 있음
- 의미적으로 비슷한 단어가 가까운 위치에 놓일 수 있음
- 머신러닝과 딥러닝 모델이 활용하기 쉬움

예를 들어:

- `good`
- `great`
- `excellent`

같은 단어들은 분산 표현 공간에서 가까운 벡터를 갖도록 학습될 수 있다.

### 1.5 임베딩과의 연결

임베딩(embedding)은 바로 이런 분산 표현을 학습하는 방법이라고 볼 수 있다.

즉, 임베딩 모델의 핵심 목적은:

- 단어를 단순히 구분하는 것이 아니라
- 단어의 의미와 관계를 벡터에 담는 것

이다.

그래서 이후 등장하는:

- `Word2Vec`
- `GloVe`
- `FastText`
- `BERT`

같은 모델들은 모두 서로 다른 방식으로 텍스트의 분산 표현을 학습한다고 이해할 수 있다.

## 2. 임베딩이란 무엇인가

임베딩(embedding)은 단어나 문장 같은 텍스트를 dense vector 형태의 숫자 표현으로 바꾸는 방법이다.

전통적인 `CountVectorizer`나 `TF-IDF`는 단어의 등장 횟수 중심의 희소 벡터를 만들었다.  
반면 임베딩은 비슷한 의미를 가진 단어들이 벡터 공간에서도 가깝게 위치하도록 학습된다.

예를 들어:

- `king`
- `queen`
- `man`
- `woman`

같은 단어들은 단순 빈도만 보는 것이 아니라, 의미적 관계까지 어느 정도 반영된 벡터로 표현될 수 있다.

## 3. 임베딩 모델을 볼 때의 기준

임베딩 모델을 비교할 때는 보통 다음 질문을 기준으로 보면 좋다.

- 단어 임베딩인가, 문장 임베딩인가
- 정적(static) 임베딩인가, 문맥(contextual) 임베딩인가
- OOV(out-of-vocabulary) 문제를 어떻게 다루는가
- 학습 방식이 local context 중심인가, global statistics 중심인가
- 계산 비용과 활용 난이도는 어떤가

## 4. Word2Vec

`Word2Vec`은 가장 대표적인 단어 임베딩 모델 중 하나다.  
단어 주변의 문맥을 이용해서 단어 벡터를 학습한다.

대표적인 학습 방식은 두 가지다.

- `CBOW`
- `Skip-gram`

### CBOW

주변 단어들을 보고 중심 단어를 예측하는 방식이다.

예:

- "I love \_\_\_ very much"

같이 주변 단어를 통해 빈칸의 중심 단어를 맞히는 식이다.

### Skip-gram

중심 단어를 보고 주변 단어를 예측하는 방식이다.

예:

- 중심 단어가 `love`일 때
- 주변 단어 `I`, `very`, `much` 등을 예측하도록 학습

보통 Skip-gram은 드문 단어 표현에 조금 더 강하다고 알려져 있다.

### 특징

- 단어 간 의미 유사성을 잘 반영함
- 계산 효율이 비교적 좋음
- 대표적인 고전 임베딩 모델
- 단어 하나당 하나의 고정 벡터를 가짐

### 한계

- 문맥을 반영하지 못함
- 같은 단어라도 문장에 따라 의미가 달라지는 경우를 처리하기 어려움
- vocabulary에 없는 단어는 다루기 어려움

예를 들어 `bank`가:

- 은행
- 강둑

중 어떤 의미인지 구분하지 못하고 항상 같은 벡터를 사용한다.

## 5. GloVe

`GloVe`는 `Global Vectors for Word Representation`의 약자다.  
Word2Vec처럼 단어 임베딩을 학습하지만, 학습 아이디어가 조금 다르다.

Word2Vec이 주변 문맥 예측을 중심으로 학습한다면, GloVe는 말뭉치 전체의 단어 동시 등장(co-occurrence) 통계를 활용한다.

즉:

- Word2Vec: local context 중심
- GloVe: global statistics 중심

으로 이해할 수 있다.

### 특징

- 전체 말뭉치의 공기어(co-occurrence) 정보를 반영
- 단어 의미 관계를 잘 포착하는 편
- 고전적인 정적 단어 임베딩의 대표 모델

### 한계

- 역시 단어 하나당 하나의 고정 벡터를 사용함
- 문맥에 따라 의미가 달라지는 문제를 해결하지 못함
- OOV 문제에 취약함

즉, GloVe도 Word2Vec과 마찬가지로 정적 단어 임베딩이라는 점은 같다.

## 6. FastText

`FastText`는 Word2Vec을 확장한 형태의 임베딩 모델이다.  
가장 중요한 차이는 단어를 하나의 단위로만 보지 않고, 문자 n-gram 단위까지 활용한다는 점이다.

예를 들어 `apple`이라는 단어를 다룰 때, 내부적으로는:

- `app`
- `ppl`
- `ple`

같은 subword 조각 정보를 함께 활용할 수 있다.

### 특징

- 단어 내부 형태 정보를 활용함
- 철자 변화나 형태 변화에 비교적 강함
- OOV(Out Of Vocabulary) 문제를 완화하는 데 유리함
- 한국어처럼 형태 변화가 있는 언어에서도 장점이 있을 수 있음

### 장점

- vocabulary에 없는 단어도 subword 조각을 통해 어느 정도 벡터를 만들 수 있음
- 오타나 희귀 단어 처리에 상대적으로 강함

### 한계

- 여전히 정적 임베딩임
- 문맥에 따라 단어 의미가 달라지는 문제는 해결하지 못함

즉, FastText는 Word2Vec보다 더 유연하지만, 문맥 임베딩은 아니다.

## 7. ELMo

`ELMo`는 문맥을 반영하는 초기 딥러닝 기반 임베딩 모델 중 하나다.  
이전의 Word2Vec, GloVe, FastText와 달리, 단어 하나가 항상 같은 벡터를 갖지 않는다.

예를 들어 같은 `bank`라도:

- `I deposited money in the bank.`
- `We sat by the bank of the river.`

에서는 서로 다른 벡터로 표현될 수 있다.

### 특징

- 문맥을 반영하는 contextual embedding
- 단어 의미가 문장에 따라 달라질 수 있음을 반영
- 양방향 언어 모델을 활용

### 의의

ELMo는 "단어 하나당 하나의 벡터"라는 고전 임베딩의 한계를 넘는 중요한 전환점이었다.

## 8. BERT

`BERT`는 Transformer 기반의 대표적인 문맥 임베딩 모델이다.  
단어를 단독으로 보지 않고 문장 전체 문맥 속에서 표현한다.

### 특징

- contextual embedding
- Transformer encoder 기반
- 양방향 문맥 반영
- pretrained model로 널리 활용됨

### 장점

- 문맥 이해 능력이 뛰어남
- 같은 단어라도 문장에 따라 다른 벡터를 만들 수 있음
- 감성 분석, 분류, 질의응답 등 다양한 작업에 활용 가능

### 한계

- 계산량이 큼
- 사용이 Word2Vec, GloVe보다 복잡함
- 실습 환경에 따라 속도와 메모리 부담이 있을 수 있음

## 9. Sentence-BERT

`Sentence-BERT`는 BERT를 문장 임베딩에 더 적합하게 만든 모델이다.  
BERT 자체는 문장 전체를 잘 이해하지만, 문장 간 유사도 계산을 직접 효율적으로 하기는 불편한 경우가 있다.

Sentence-BERT는 이런 문제를 보완해서 문장을 하나의 dense vector로 표현하고, 코사인 유사도 같은 방식으로 문장 간 의미 유사도를 비교하기 쉽게 만든다.

### 특징

- 문장 임베딩에 특화
- 의미 기반 문장 유사도 계산에 유리
- 검색, 클러스터링, 추천, semantic search 등에 자주 사용

### 활용 예

- 비슷한 문장 찾기
- FAQ 매칭
- 문서 검색
- 문장 군집화

## 10. 정적 임베딩과 문맥 임베딩의 차이

대표적인 차이는 다음과 같다.

### 정적 임베딩

예:

- Word2Vec
- GloVe
- FastText

특징:

- 단어 하나당 하나의 고정 벡터
- 계산이 비교적 단순함
- 고전적인 NLP 실습에 자주 사용
- 문맥을 반영하지 못함

### 문맥 임베딩

예:

- ELMo
- BERT
- Sentence-BERT

특징:

- 문장 맥락에 따라 벡터가 달라짐
- 의미 표현이 더 풍부함
- 현대 NLP에서 더 널리 활용됨
- 계산 비용이 더 큼

## 11. 모델별 비교표

한눈에 비교하면 다음과 같다.

| 모델 | 단위 | 문맥 반영 | 강점 | 한계 |
| --- | --- | --- | --- | --- |
| `Word2Vec` | 단어 | 아니오 | 가볍고 직관적이며 고전 임베딩 이해에 좋음 | OOV와 다의어 처리에 약함 |
| `GloVe` | 단어 | 아니오 | 전역 공기어 통계를 반영해 의미 관계 포착에 강함 | 정적 임베딩이라 문맥 차이를 반영하지 못함 |
| `FastText` | 단어 + subword | 아니오 | 철자 변화, 희귀 단어, OOV 대응이 상대적으로 좋음 | 여전히 정적 임베딩이라 문맥 차이를 반영하지 못함 |
| `ELMo` | 단어 | 예 | 문맥에 따라 같은 단어를 다르게 표현할 수 있음 | 최근 기준으로는 활용 사례가 줄고 구조 이해용 성격이 강함 |
| `BERT` | 토큰/문장 문맥 | 예 | 다양한 NLP 태스크에 강력하고 범용성이 높음 | 계산 비용이 크고 실습 난이도가 높음 |
| `Sentence-BERT` | 문장 | 예 | 문장 유사도, 검색, semantic search에 특히 적합함 | 문장 임베딩 목적에는 강하지만 모든 태스크의 기본 해법은 아님 |

## 12. 어떤 모델을 언제 쓰는가

학습 단계에서는 보통 다음처럼 감을 잡으면 된다.

- 고전적인 단어 임베딩 개념을 이해하고 싶을 때
  - `Word2Vec`, `GloVe`

- 희귀 단어나 형태 정보까지 조금 더 보고 싶을 때
  - `FastText`

- 문맥에 따라 단어 의미가 달라지는 것을 반영하고 싶을 때
  - `ELMo`, `BERT`

- 문장 단위 의미 비교가 필요할 때
  - `Sentence-BERT`

즉, "단어를 표현하고 싶은가", "문장을 표현하고 싶은가", "문맥이 중요한가"에 따라 선택이 달라진다.

## 13. 감성 분석과의 연결

감성 분석에서는 어떤 임베딩을 쓰느냐에 따라 접근 방식이 달라진다.

전통적인 방식에서는:

- `CountVectorizer`
- `TF-IDF`
- 또는 정적 단어 임베딩

을 사용한 뒤, 별도의 분류기를 붙이는 경우가 많다.

반면 딥러닝 기반 감성 분석에서는:

- pretrained embedding
- contextual embedding
- BERT 계열 모델

을 사용해 문장 전체 문맥을 반영한 분류를 수행할 수 있다.

특히 감성 분석에서는:

- `not good`
- `very good`
- `hardly useful`

처럼 문맥과 조합이 중요한 표현이 많기 때문에, 문맥 임베딩의 장점이 더 크게 드러날 수 있다.

## 14. 핵심 요약

- 임베딩은 텍스트를 dense vector로 바꾸어 의미 관계를 표현하려는 방법이다.
- `Word2Vec`, `GloVe`, `FastText`는 정적 임베딩이라 단어마다 하나의 고정 벡터를 갖는다.
- `ELMo`, `BERT`, `Sentence-BERT`는 문맥을 반영해 같은 단어도 상황에 따라 다르게 표현할 수 있다.
- 단어 수준 표현이 필요한지, 문장 수준 표현이 필요한지에 따라 적합한 모델이 달라진다.
- 감성 분석, 검색, 유사도 계산처럼 문맥이 중요한 작업일수록 문맥 임베딩의 장점이 커진다.

## 15. 정리

임베딩 모델들은 모두 텍스트를 벡터로 바꾼다는 공통점이 있지만, 표현 방식과 반영하는 정보 수준이 다르다.

핵심 흐름을 정리하면 다음과 같다.

- `Word2Vec`, `GloVe`: 고전적인 정적 단어 임베딩
- `FastText`: subword 정보를 반영한 정적 단어 임베딩
- `ELMo`, `BERT`: 문맥을 반영하는 임베딩
- `Sentence-BERT`: 문장 임베딩에 특화된 모델

즉, 임베딩 모델의 발전 흐름은 대체로 다음처럼 볼 수 있다.

- 단어 빈도 기반 표현
- 정적 단어 임베딩
- 문맥 기반 임베딩
- 문장 임베딩

이 흐름을 잡아두면 이후 감성 분석, 문장 유사도, 검색, 분류 같은 여러 NLP 작업에서 왜 어떤 모델을 선택하는지 이해하기 쉬워진다.
