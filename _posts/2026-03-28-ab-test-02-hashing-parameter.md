# A/B 테스트에서 `hashing parameter`란?

## 1. `hashing parameter`란 무엇인가

A/B 테스트에서 `hashing parameter`는  
"어떤 사용자를 어떤 bucket(버킷)에 넣을지 결정할 때 기준이 되는 값"이다.

보통 아래 같은 식별값을 사용한다.

- `user_id`
- `device_id`
- `cookie_id`

즉, 사용자를 실험군(A/B)에 나눌 때 기준이 되는 키라고 이해하면 된다.

## 2. 여기서 `hashing`이란 무슨 뜻인가

여기서 `hashing`은 `user_id` 같은 값을 해시 함수에 넣어서  
일정한 규칙을 가진 숫자값으로 바꾸는 과정을 말한다.

예를 들어:

- `user_101`
- `user_102`
- `user_103`

같은 문자열이 있으면, 해시 함수는 각각 다른 숫자 같은 결과를 만들어낸다.

예:

- `hash("user_101") -> 3842`
- `hash("user_102") -> 9121`
- `hash("user_103") -> 1044`

이 숫자를 이용해 사용자를 A 또는 B 버킷에 배정한다.

## 3. 왜 굳이 해시를 쓰는가

핵심 이유는 같은 사용자를 항상 같은 variant에 넣기 위해서다.

예를 들어 어떤 사용자가:

- 오늘은 A를 보고
- 내일은 B를 보고
- 모레는 또 A를 보면

실험이 섞여서 결과 해석이 어려워진다.

그래서 보통은 같은 입력값이면 항상 같은 결과를 내는 해시 함수를 사용해  
사용자를 안정적으로 같은 버킷에 넣는다.

해시를 쓰면 보통 아래 장점이 있다.

- 같은 사용자에게 항상 같은 배정 결과를 줄 수 있다.
- 사람이 보기에는 랜덤처럼 고르게 분산된다.
- 빠르게 버킷을 나눌 수 있다.

## 4. 버킷 배정은 어떻게 하나

보통은 해시 결과를 일정 범위의 숫자로 바꾼 다음,  
그 숫자를 기준으로 버킷을 나눈다.

예를 들어:

```python
bucket = hash(user_id) % 100
```

라고 하면 `0`부터 `99` 사이 숫자가 나온다고 생각할 수 있다.

그다음 이렇게 나눌 수 있다.

- `0~49`: A 버킷
- `50~99`: B 버킷

즉:

```python
if bucket < 50:
    variant = "A"
else:
    variant = "B"
```

이렇게 하면 대략 50:50으로 사용자가 나뉜다.

## 5. 10%만 테스트하고 싶다면?

예를 들어 새 기능을 전체 사용자 중 10%에게만 노출하고 싶다면  
아래처럼 생각할 수 있다.

- `0~89`: Control
- `90~99`: Test

즉:

```python
bucket = hash(user_id) % 100

if bucket < 90:
    variant = "control"
else:
    variant = "test"
```

그러면 전체 사용자 중 대략 10%만 테스트 버전에 들어간다.

## 6. 왜 `user_id`, `device_id`, `cookie_id`를 다르게 쓸 수 있나

서비스 상황에 따라 기준 키가 달라질 수 있다.

### `user_id`

로그인 기반 서비스에서 자주 사용한다.

장점:

- 같은 사용자를 안정적으로 추적하기 쉽다.

예:

- 쇼핑몰 회원
- OTT 구독자
- 로그인 사용자 중심 서비스

### `device_id`

앱 환경에서 로그인하지 않아도 어느 정도 기기를 기준으로 나누고 싶을 때 사용한다.

예:

- 모바일 앱
- 비로그인 상태 사용자 실험

### `cookie_id`

웹에서 비로그인 사용자 실험을 할 때 자주 사용한다.

예:

- 랜딩 페이지 실험
- 비회원 유입 실험

## 7. `hashing parameter`가 중요한 이유

이 값이 잘못 선택되면 실험도 꼬일 수 있다.

예:

- 로그인 사용자인데 세션 ID를 기준으로 나누면, 같은 사람이 접속할 때마다 다른 버전에 들어갈 수 있다.
- 디바이스를 여러 개 쓰는 사용자는 `device_id` 기준일 때 서로 다른 경험을 할 수 있다.

즉, 실험 목적에 맞는 기준 키를 선택해야 한다.

## 8. 쉬운 예시

예를 들어 이커머스에서 추천 알고리즘 A/B 테스트를 한다고 하자.

- 기준 키: `user_id`
- 버킷 방식: `hash(user_id) % 100`
- `0~49`: 기존 추천(A)
- `50~99`: 새 추천(B)

그러면:

- 사용자 1은 항상 A
- 사용자 2는 항상 B
- 사용자 3은 항상 A

처럼 안정적으로 배정된다.

이렇게 해야 추천 클릭률, 구매율, 매출을 그룹별로 비교할 수 있다.

## 9. 한 줄 정리

`hashing parameter`는 사용자 식별값(`user_id`, `device_id` 등)을 해시해  
각 사용자를 A/B 버킷에 안정적이고 균등하게 배정하기 위한 기준값이다.

## 10. 주석처럼 짧게 적으면

```md
- Hashing parameter: user_id/device_id 같은 식별값을 해시해 사용자를 A/B 버킷에 안정적으로 배정하는 기준
```
