# A/B 테스트 통계 검정 실습 해설

## 1. 이 문서는 무엇을 설명하는가

이 문서는 [A_B_Test_03강_통계_검정_실습.ipynb](/assets/files/notebooks/ab_test/A_B_Test_03%EA%B0%95_%ED%86%B5%EA%B3%84_%EA%B2%80%EC%A0%95_%EC%8B%A4%EC%8A%B5.ipynb) 흐름을 설명하는 해설 노트다.

이 노트북은 실제 서비스 로그를 바로 분석하는 실습이 아니라,  
`A와 B가 실제로 다르다`는 상황을 인위적으로 만든 뒤  
샘플 데이터만 보고도 그 차이를 통계적으로 잡아낼 수 있는지 이해하는 데 목적이 있다.

즉, 이 실습은 아래 질문에 답하기 위한 것이다.

- 실제로 B가 더 좋은 상황이라면 샘플에서는 어떻게 보일까
- z-test는 그 차이를 어떻게 계산할까
- p-value와 신뢰구간은 무엇을 말해줄까
- 같은 조건에서도 샘플이 바뀌면 결과는 얼마나 흔들릴까

## 2. 노트북 전체 흐름 한눈에 보기

노트북 흐름은 크게 일곱 단계다.

1. 실험 환경 설정
2. 샘플 데이터 생성
3. 관측 전환율 계산
4. two-proportion z-test 수행
5. 95% 신뢰구간 계산
6. 결과 해석
7. 같은 실험을 여러 번 반복해 분포 확인

즉, 이 노트북은  
`실제 차이가 있는 세계를 가정 -> 샘플 생성 -> 검정 수행 -> 결과 해석 -> 반복 실험으로 흔들림 확인`  
순서로 구성되어 있다.

## 3. 실험 설정 셀은 무엇을 하는가

노트북 초반에는 아래 값을 정한다.

```python
true_p_a = 0.10
true_p_b = 0.12
```

이 값은 매우 중요하다.

이건 우리가 현실에서 이미 아는 값이 아니라,  
실습을 위해 `정답을 심어놓은 값`이다.

즉:

- A의 실제 전환 확률은 10%
- B의 실제 전환 확률은 12%

인 세계를 인위적으로 만든 것이다.

그래서 이 노트북은  
실무 데이터 분석이라기보다  
`실제 차이가 있는 상황에서 통계 검정이 어떻게 반응하는지 보는 시뮬레이션`에 가깝다.

## 4. 왜 `true_p_a`, `true_p_b`를 미리 넣는가

실무에서는 보통 실제 전환율을 모른다.

우리가 아는 것은:

- control 사용자 수
- test 사용자 수
- 전환 수
- 관측 전환율

뿐이다.

반면 이 실습은 학습용이기 때문에  
일부러 실제 값을 먼저 정해놓는다.

그 이유는:

- 진짜 차이가 있는 상황을 만들고
- 샘플로 보면 그 차이가 어떻게 흔들리는지 보고
- z-test와 p-value가 어떤 식으로 반응하는지 이해하기 위해서다

즉, `true_p_a`, `true_p_b`는  
현실 데이터가 아니라 시뮬레이션의 정답지라고 보면 된다.

## 5. 데이터 생성 셀은 무엇을 뜻하나

노트북에서는 아래처럼 데이터를 만든다.

```python
a = rng.binomial(1, true_p_a, size=n_a)
b = rng.binomial(1, true_p_b, size=n_b)
```

이 코드는 각 사용자가:

- 전환하면 `1`
- 전환하지 않으면 `0`

이 되도록 A와 B 그룹 사용자 데이터를 만든다.

즉:

- `a`는 A 그룹 사용자들의 전환 결과 배열
- `b`는 B 그룹 사용자들의 전환 결과 배열

이다.

여기서 `size=n_a`, `size=n_b`는 사용자 수를 의미한다.

## 6. 관측 전환율 계산 셀은 무엇을 보여주나

다음 셀에서는:

```python
obs_p_a = a.mean()
obs_p_b = b.mean()
diff = obs_p_b - obs_p_a
```

를 계산한다.

여기서 중요한 개념은  
`실제 전환율`과 `관측 전환율`의 차이다.

### 실제 전환율

우리가 시뮬레이션 세계에 심어놓은 진짜 값이다.

예:

- `true_p_a = 0.10`
- `true_p_b = 0.12`

### 관측 전환율

샘플 데이터에서 계산한 값이다.

예:

- 어떤 실험에서는 `obs_p_a = 0.097`
- `obs_p_b = 0.118`

처럼 나올 수 있다.

즉, 샘플에는 항상 우연이 섞이므로  
관측 전환율은 실제 전환율과 완전히 같지 않을 수 있다.

이 셀의 핵심은  
우리가 실제로 검정에 사용하는 값은 `true_p_*`가 아니라 `obs_p_*`라는 점이다.

## 7. z-test 셀은 무엇을 하는가

노트북의 중심은 `two_proportion_z_test(...)` 함수다.

이 함수는  
두 비율 차이가 우연만으로 설명 가능한지 계산한다.

귀무가설은 다음과 같다.

- `H0: p_A = p_B`

즉, A와 B의 전환율이 같다고 놓고 시작한다.

### `pooled_p`는 무엇인가

```python
pooled_p = (x_a + x_b) / (n_a + n_b)
```

귀무가설 아래에서는 두 그룹 비율이 같다고 보기 때문에,  
두 그룹을 합쳐 하나의 공통 전환율을 추정한다.

이 값이 `pooled_p`다.

### `se`는 무엇인가

```python
se = math.sqrt(pooled_p * (1 - pooled_p) * (1 / n_a + 1 / n_b))
```

이 값은 `표준오차`다.

표준오차는  
`우연 때문에 비율 차이가 어느 정도 흔들릴 수 있는가`를 나타낸다.

즉:

- 표준오차가 크면 차이가 많이 흔들릴 수 있고
- 표준오차가 작으면 작은 차이도 더 의미 있어진다

### `z-score`는 무엇인가

```python
z = (p_b - p_a) / se
```

관측된 차이 `B-A`가  
표준오차에 비해 몇 배나 큰지 계산한 값이다.

즉:

- z-score가 0 근처면 흔한 차이
- z-score가 아주 크거나 아주 작으면 드문 차이

라고 이해하면 된다.

### `p-value`는 무엇인가

```python
p_value = 2 * (1 - normal_cdf(abs(z)))
```

이 값은  
귀무가설이 참이라고 했을 때  
지금 같은 차이 또는 그보다 더 극단적인 차이가 나올 확률이다.

즉:

- p-value가 크면 우연으로도 충분히 나올 수 있음
- p-value가 작으면 우연만으로 보기 어려움

이다.

## 8. 신뢰구간 셀은 무엇을 말해주나

다음 셀에서는:

```python
ci_low = diff - 1.96 * se_diff
ci_high = diff + 1.96 * se_diff
```

로 95% 신뢰구간을 계산한다.

여기서 보는 값은 전환율 자체가 아니라  
`B-A 차이`다.

즉, 이 셀은  
`B와 A의 차이가 어느 범위 안에 있을 것 같은가`  
를 추정한다.

## 9. 여기서 왜 `0`이 중요한가

이 부분이 가장 자주 헷갈린다.

우리가 지금 보고 있는 값은 `B-A`다.

그러면:

- `B-A = 0`

은 무엇을 뜻할까?

바로 `차이가 없다`는 뜻이다.

즉:

- 귀무가설 `A와 B는 같다`
- 차이 기준으로 쓰면 `B-A = 0`

이 된다.

그래서 신뢰구간을 보고  
`0이 이 범위 안에 들어가는가`를 확인하는 것이다.

### 0이 신뢰구간 안에 있으면

차이가 없다는 설명도 여전히 가능하다.

즉:

- A와 B가 같을 수도 있고
- 우연 때문에 지금처럼 보였을 수도 있다

는 뜻이다.

### 0이 신뢰구간 밖에 있으면

차이가 없다는 설명과 잘 맞지 않는다.

즉:

- B-A가 실제로 0이 아닐 가능성이 높고
- A와 B 차이가 있다고 볼 근거가 커진다

는 뜻이다.

## 10. 결과 해석 셀은 무엇을 하는가

결과 해석 셀에서는:

- `p_value < 0.05`인지
- `diff > 0`인지

를 같이 본다.

즉 두 가지를 동시에 확인하는 셀이다.

### 1. 통계적으로 유의한가

- 귀무가설을 기각할 수 있는가

### 2. 방향이 어디인가

- B가 A보다 좋은가
- 아니면 오히려 나쁜가

즉, 유의성만 보는 것이 아니라  
방향성도 같이 확인하는 단계다.

## 11. 왜 같은 실험을 여러 번 반복하나

한 번의 실험 결과는 우연이 섞일 수 있다.

같은 실제 전환율 조건에서도:

- 어떤 실험에서는 차이가 크게 보이고
- 어떤 실험에서는 작게 보이고
- 어떤 실험에서는 p-value가 더 작게 나오고
- 어떤 실험에서는 경계선 근처가 될 수 있다

그래서 반복 시뮬레이션은  
`관측 결과가 얼마나 흔들리는가`를 보여주는 역할을 한다.

즉, 이 파트는  
한 번의 결과만 보고 너무 확신하면 안 된다는 점을 몸으로 이해하게 해준다.

## 12. 반복 시뮬레이션 셀은 무엇을 저장하나

반복 루프에서는 매 실험마다 아래를 저장한다.

- `diffs`
- `p_values`

### `diffs`

각 반복 실험에서 나온 `B-A 차이`다.

즉, 실제 차이가 0.02인 상황에서도  
샘플에서는 0.015, 0.023, 0.011처럼 흔들릴 수 있다는 걸 보여준다.

### `p_values`

각 반복 실험에서 계산된 p-value다.

즉, 같은 실제 조건에서도  
p-value는 매번 조금씩 달라질 수 있다.

## 13. 첫 번째 그래프는 무엇을 보여주나

첫 번째 그래프는  
반복 실험에서 관측된 `B-A 차이`의 분포를 보여준다.

여기서 보는 핵심은:

- 0 근처가 아니라 실제 차이 근처에 분포가 모이는지
- 샘플 차이가 얼마나 흔들리는지

이다.

그래프에는 두 기준선이 있다.

- `0`: 차이 없음
- `true_p_b - true_p_a`: 실제로 심어놓은 진짜 차이

즉, 샘플 차이들이  
차이 없음이 아니라 실제 차이 주변에서 흔들리는지를 보는 그래프다.

## 14. 두 번째 그래프는 무엇을 보여주나

두 번째 그래프는  
반복 실험에서 계산된 p-value 분포를 보여준다.

여기서 보는 핵심은:

- 0.05보다 작은 p-value가 얼마나 자주 나오는가

이다.

즉, 실제로 차이가 있는 상황에서  
검정이 그 차이를 얼마나 자주 유의하게 잡아내는지 보는 그래프라고 할 수 있다.

## 15. 이 노트북에서 정말 가져가야 할 핵심

- `true_p_a`, `true_p_b`는 학습용으로 심어놓은 실제 모수다.
- 우리가 실제 검정에 쓰는 값은 `obs_p_a`, `obs_p_b` 같은 관측값이다.
- z-test는 관측 차이를 표준오차로 나누어 `얼마나 드문 차이인지`를 계산한다.
- p-value는 그 차이가 우연으로 설명될 가능성을 나타낸다.
- 신뢰구간에서 `0`이 중요하다는 것은 `B-A=0`이 곧 차이 없음이기 때문이다.
- 반복 시뮬레이션은 샘플 결과가 얼마나 흔들릴 수 있는지 보여준다.

## 16. 자주 헷갈리는 질문

### 실무에서도 `true_p_a`, `true_p_b`를 아는가

보통 모른다.

실무에서는 관측 전환율만 알고,  
실제 전환율은 추론 대상이다.

### 왜 정답을 먼저 넣고 시작하나

학습용 시뮬레이션이기 때문이다.

실제로 차이가 있는 상황을 일부러 만든 뒤,  
통계 검정이 그 차이를 어떻게 잡는지 보기 위해서다.

### 관측 전환율과 실제 전환율은 왜 다른가

샘플에는 표본 오차가 있기 때문이다.

그래서 관측 전환율은 실제 전환율의 추정치일 뿐이다.

### 신뢰구간에서 0은 왜 중요한가

우리가 보는 값이 `B-A`라서,  
0은 곧 `차이 없음`을 뜻한다.

## 17. 한 줄 요약

이 노트북은  
실제 차이가 있는 A/B 테스트 상황을 인위적으로 만들고,  
샘플에서 관측한 차이만 가지고 z-test, p-value, 신뢰구간으로  
그 차이를 얼마나 잘 추론할 수 있는지 이해하는 실습 노트북이다.
