# A/B Test 01강 퀴즈 복습노트

## 1. A/B 테스트를 시작함에 있어서 가장 중요한 것은?

문항:

- 가설을 세우는 것
- 빠르게 속도를 내서 실험을 하는 것

정답:

- `가설을 세우는 것`

이유:

- A/B 테스트는 아무 아이디어나 던져보는 작업이 아니라, 가설을 검증하는 실험이다.
- 무엇을 바꾸는지, 어떤 지표가 바뀔지, 어느 방향으로 얼마나 변할지를 먼저 정해야 결과를 해석할 수 있다.
- 속도도 중요하지만, 가설 없이 빠르게만 돌리면 학습이 남지 않는다.

핵심 복습:

- A/B 테스트의 출발점은 `속도`보다 `가설`이다.

## 2. Agile한 A/B 테스트가 필요한 이유는?

예시 답안:

애자일한 A/B 테스트가 필요한 이유는 실험 하나를 너무 오래 끌면 제품 개선 속도와 출시 속도가 같이 느려지기 때문이다.  
실험 설정, 분석, 배포를 며칠~몇 주 단위로 빠르게 반복할 수 있어야 더 빨리 배우고 더 빨리 좋은 기능을 출시할 수 있다.

짧게 쓰면:

- `실험 시간을 줄여 더 빠르게 학습하고 더 빠르게 제품을 개선하기 위해서`

핵심 복습:

- Agile A/B Test = 빠른 실험 반복 = 빠른 제품 개선

## 3. A/B 테스트 가설에 필요하지 않은 부분은?

문항:

- What pain point or opportunity?
- Expected Impact
- Metrics to determine success/failure
- Who will analyze the A/B test results

정답:

- `Who will analyze the A/B test results`

이유:

- 가설에는 보통 문제 정의, 기대 효과, 성공/실패 판단 지표가 들어간다.
- 반면 누가 분석할지는 운영/역할 분담 정보이지, 가설 자체의 필수 구성요소는 아니다.

핵심 복습:

- 가설에는 `문제`, `예상 효과`, `성공 지표`가 중요하다.
- `분석 담당자`는 실험 운영 정보에 가깝다.

## 4. A/B 테스트하기에 적합한 것은?

선택지:

- 버그 수정으로 인한 매출액 증대
- 아직 구체적이지 않은 초기 아이디어
- 간밤에 생각난 재미있는 아이디어
- 가설을 바탕으로 한 구체적인 프로덕트 개선 아이디어

정답:

- `가설을 바탕으로 한 구체적인 프로덕트 개선 아이디어`

이유:

- 버그 수정은 실험보다 빠른 수정이 우선이다.
- 아직 구체적이지 않은 초기 아이디어는 먼저 인터뷰, 설문, 프로토타입 검증이 더 적합하다.
- 즉흥적이고 가설 없는 아이디어도 A/B 테스트에 적합하지 않다.
- 반면 가설을 바탕으로 한 구체적인 개선안은 비교 대상이 분명하고, 어떤 지표를 볼지도 정하기 쉬워 A/B 테스트에 적합하다.

핵심 복습:

- A/B 테스트는 `가설이 있고`, `비교 대상이 있고`, `측정 가능한 변화`에 적합하다.

## 5. A/B 테스트에서 생길 수 있는 Interaction은 무엇인가?

예시 답안:

여러 A/B 테스트를 동시에 돌릴 때 서로 다른 실험이 같은 사용자 행동이나 KPI에 영향을 주면서 결과가 섞이는 현상이다.

예:

- 장바구니 페이지 실험
- 체크아웃 페이지 실험

을 동시에 돌리면 구매율 변화가 장바구니 때문인지 체크아웃 때문인지 구분하기 어려울 수 있다.

짧게 쓰면:

- `동시에 진행된 여러 실험이 서로 영향을 주어 결과 해석을 어렵게 만드는 것`

핵심 복습:

- Interaction = 실험 간 간섭

## 6. 상품 추천 관련 A/B 테스트 분석을 하는데 필요하지 않은 데이터가 아닌 것은?

문항:

- 사용자 A/B 소속 정보 (bucketing data)
- 추천 상품 사용자별 Funnel 정보 (impression, click, purchase)
- 마케팅 채널 정보

질문의 의도:

- "분석에 꼭 필요한 기본 데이터가 무엇인가?"를 묻는 문제로 보인다.

기본적으로 꼭 필요한 데이터:

- `사용자 A/B 소속 정보 (bucketing data)`
- `추천 상품 사용자별 Funnel 정보 (impression, click, purchase)`

상대적으로 필수는 아닌 데이터:

- `마케팅 채널 정보`

이유:

- A/B 분석의 최소 조건은 "누가 A/B에 속했는지"와 "그 사용자가 어떤 행동을 했는지"다.
- 마케팅 채널 정보는 세그먼트 분석에는 유용하지만, 기본 분석의 필수 조건은 아니다.

핵심 복습:

- 최소 필수 데이터는 `버킷 데이터 + 행동(funnel) 데이터`다.

## 7. 꼭 기억할 핵심

- A/B 테스트는 가설을 검증하는 실험이다.
- 빠른 실험도 중요하지만, 가설 없는 실험은 의미가 약하다.
- Agile한 실험 운영은 제품 개선 속도를 높인다.
- 가설에는 문제 정의, 기대 효과, 성공 지표가 필요하다.
- 버그 수정, 초기 아이디어, 즉흥적 아이디어는 바로 A/B 테스트하기에 부적절하다.
- 여러 실험을 동시에 돌리면 interaction이 생길 수 있다.
- 분석의 최소 조건은 `누가 어느 버킷에 들어갔는지`와 `그 사용자가 무엇을 했는지`를 아는 것이다.

## 8. 한 줄 요약

A/B 테스트의 핵심은  
`가설을 기반으로 실제 사용자 데이터를 비교하고, 이를 빠르고 안전하게 운영하며, 정확한 버킷/행동 데이터로 해석해야 한다`는 것이다.
